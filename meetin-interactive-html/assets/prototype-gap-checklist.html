建议优先补的页面

  • 运营后台：`待审核列表`
  • 运营后台：`内容审核`
  • 运营后台：`发布确认`
  • 运营后台：`结算管理`
  • 通用：`消息中心`
  • 广告主端：`任务审核结果 / 驳回原因`
  • 达人端：`入驻认证`、`发布链接提交`、`版本记录`

注释模式建议

  • 页面右上角增加 `查看说明`，点击后打开右侧说明抽屉。
  • 复杂模块旁增加小型 `i` 信息点，点击看局部规则。
  • 默认隐藏说明，不影响主流程操作体验。
  • 注释内容建议固定为：`页面目标 / 区块说明 / 状态规则 / 后续待补`。
备注

当前原型的强项是广告主端和达人端主链路已经具备演示基础；下一轮建议重点把运营后台与审核/发布/结算闭环补齐，这样更符合这份 PRD 的 V1.0 评审需求。
